\# Security Baseline Compliance Methodology



\## Overview



This project validates Windows and Linux systems against a basic security baseline.



The scanner checks key security controls, assigns pass/fail/warn/info status, calculates a compliance score, and generates remediation guidance.



\## Control Categories



| Category | Purpose |

|---|---|

| Firewall | Validate host-level network protection |

| Endpoint Monitoring | Confirm security agents are running |

| Audit Logging | Validate visibility and forensic readiness |

| SSH Security | Reduce unauthorized Linux access risk |

| Password Policy | Reduce credential attack risk |

| Privileged Access | Review administrative access |

| Remote Access | Identify remote access exposure |



\## Scoring Model



Controls are scored as:



| Status | Meaning |

|---|---|

| PASS | Control is configured as expected |

| FAIL | Control is missing or misconfigured |

| WARN | Control requires review or improvement |

| INFO | Evidence collected for manual review |



\## Security Relevance



Security baselines help validate whether systems meet expected hardening and monitoring requirements. This supports risk reduction, audit readiness, and continuous security improvement.
